1. Introduction to Git

Git is a Distributed Version Control System (DVCS). It helps developers track code changes, collaborate with teams, and maintain project history.

Git was created by Linus Torvalds in 2005 and is widely used in software development.

2. Why Git is Used?

  • Track Code Changes
  • Team Collaboration
  • Version Management
  • Backup Source Code
  • Restore Previous Versions
  • Manage Multiple Features

Without Git, managing large software projects becomes difficult and risky.

3. Git vs GitHub

Git GitHub
Version Control Tool Cloud Hosting Platform
Works Locally Works Online
Tracks Changes Stores Repositories
Installed on System Website Platform

4. Installing Git

Download Git from the official website and install it on your system. 

https://git-scm.com

After installation verify Git: 

git --version

5. Git Configuration

Set username and email before using Git. 

git config --global user.name "Awanish"

git config --global user.email "example@gmail.com"

Check configuration: 

git config --list

6. Create Repository

Repository is a project folder managed by Git. 

mkdir myproject

cd myproject

git init

This creates a new Git repository.

7. Important Git Commands

Command Purpose
git init Create Repository
git status Check Status
git add . Add Files
git commit -m "msg" Create Commit
git log View History

8. Add and Commit

Git follows a staging process. 

git add .

git commit -m "Initial Commit"

Commit saves project snapshot permanently.

9. Connect GitHub Repository

Create repository on GitHub and connect it. 

git remote add origin
https://github.com/username/project.git

Check remote: 

git remote -v

10. Push Code to GitHub

git branch -M main

git push -u origin main

Your project is now uploaded to GitHub.

11. Pull Changes from GitHub

Download latest changes from remote repository. 

git pull origin main

12. Branches

Branches allow developers to work on features independently. 

git branch feature-login

git checkout feature-login

List all branches: 

git branch

13. Merge Branches

Merge combines feature branch into main branch. 

git checkout main

git merge feature-login

14. Complete GitHub Workflow

Create Project

↓
git init

↓
git add .

↓
git commit

↓
Create GitHub Repo

↓
git remote add origin

↓
git push origin main

15. Deploy Website Using Netlify

Netlify is a free hosting platform used to deploy frontend projects.

Deployment Steps
  1. Create GitHub Repository
  2. Push Project to GitHub
  3. Login to Netlify
  4. Click Add New Site
  5. Import From GitHub
  6. Select Repository
  7. Deploy Site

Netlify automatically generates a live website URL.

16. Custom Domain Setup

Netlify allows connecting custom domains.

  1. Open Site Settings
  2. Go To Domains
  3. Add Custom Domain
  4. Configure DNS Records

Example: 

www.myportfolio.com
